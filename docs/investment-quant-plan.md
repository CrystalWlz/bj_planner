# 港股通 / QDII ETF 量化定投一期

## 已实现边界

一期是“研究、提案、模拟成交、人工确认”的家庭理财辅助功能，不是自动交易系统。系统只支持用户手工维护的 ETF/基金标的池，不会推荐具体基金代码，不接收或保存任何券商、同花顺或 Tushare 凭据。

- 数据源：Tushare Pro 日频数据。令牌只能读取本机 `TUSHARE_TOKEN` 环境变量。
- 标的范围：境内 ETF、已确认合资格的港股通标的、场内跨境 QDII ETF、场外 QDII 基金。
- 交易边界：一期只有 `PaperBrokerAdapter`；`QmtBrokerAdapter` 是不可执行的占位接口。
- 风险边界：可投资现金先扣除应急金、未来 24 个月计划支出和同期重大目标资金；权益上限默认 35%，防御资产下限默认 65%。

## 稳定策略接口

提案流程拆为四个可替换但默认固定的职责：`SignalModel` 只输出月度回撤风险状态；`PortfolioConstructor` 只把权益预算分配给用户批准的标的；`PreTradeRiskManager` 负责现金、交易资格、QDII 溢价/净值可得性、单标的/单市场集中度和最大订单金额；`ExecutionPlanner` 负责下一交易日、滑点、费率和最小交易单位。每个组件都有独立版本号并进入提案和研究运行记录。

一期默认只启用“月度定投 + 35/65 + 回撤降风险”。最小方差仅在 `research_strategy=min_variance` 时作为研究组件运行，默认关闭；趋势、波动率目标等策略尚未进入生产提案。

用户可把手工核实的货币/短债/债券 ETF 标记为防御资产。`PortfolioConstructor` 在权益与防御资产池之间按有效权益上限分配；没有防御标的或集中度上限未用满的部分继续保留为模拟现金，不会为了凑满 65% 自动选择产品。

季度月份仅在当前模拟权益比例偏离目标至少 5 个百分点时触发再平衡。超配持仓先生成整手卖出订单；卖出成交后，后续提案才允许使用已到账模拟现金补充低配资产。模拟现金不足或卖出超过持仓时 API 返回冲突，不会透支、截断或自动补仓。

## 策略规则

每月按最近已经确认的日线生成一次提案，季度偏离目标配置 5 个百分点才提示再平衡。风险篮子回撤达到 8% 时新增权益上限降为 20%；达到 12% 时暂停新增权益；达到 15% 时冻结风险资产提案并要求人工复核。该机制不能保证市场跳空时的实际最大回撤。

场内跨境 QDII ETF 必须同时拥有最新净值和收盘价：净值超过 3 个交易日或溢价高于策略阈值时暂停新增买入。场外 QDII 只记录净值、限购/暂停状态和人工确认，不产生交易所模拟订单。

回测使用至少三年共同可用的复权日线，并按沪深 `trade_cal`、港股 `hk_tradecal` 返回的共同开放日推进：上一个交易日生成信号，下一个所有标的均可交易的日期成交。交易日历接口无权限或不可用时，数据集会明确记录警告并回退到实际价格日期；日历开放但缺少价格的资产按停牌沿用上一估值，禁止在该日成交。停牌、申购限制、最小交易单位、费率、滑点、QDII 净值可得日期、防御资产和季度再平衡都进入撮合，不再按每 21 根日线机械压缩。输出同时包含终值、CAGR、年化波动率、最大回撤、换手率、累计费用、最低模拟现金和成交笔数；比较风险控制策略、35/65 静态配置、纯现金定投和 100% 权益固定定投。

运行记录按默认 24 个月训练窗口、12 个月测试窗口输出滚动样本外分段。默认固定规则不拟合参数，窗口只隔离评价区间；启用 `research_strategy=min_variance` 时，每个信号日只截取当日及之前最近 24 个月的权益价格重估权重，再用于下一共同可交易日，测试区间和未来数据不会进入当次拟合。该研究开关默认关闭，回测不自动选择标的，也不构成收益承诺。

## 可追溯行情数据集

每个行情版本保存数据源、接口名、抓取时间、交易日历、复权口径、价格日期、净值估值日期、净值可得日期、完整性、内容哈希和数据版本。SQLite 按 `instrument_id + dataset_hash` 保留版本；相同内容幂等复用，内容变化不会覆盖旧版本。QDII 缺少公告日期时保守假定净值最早在估值日后一天可得，当日决策不能使用尚未发布的净值。

## 模拟账户与研究记录

模拟订单只是交易意图；每个订单拥有唯一 `client_order_id`。点击模拟成交后，订单状态和成交事件必须在同一数据库事务中写入。同一订单只允许产生一条成交，重复点击返回已有成交，不重复增加持仓。月度订单预算先进入模拟现金，整手成交后的未使用预算继续保留；持仓、平均成本、市值、手续费、已实现/未实现盈亏全部由成交事件与最新行情在后端派生，不修改家庭真实现金或真实投资账户。

`PaperPortfolioLedger` 把成交事件转换成现有 `MonthlyLedgerEntry → AccountSnapshotPoint → MonthlyVisualizationDetail` 数据结构，并作为 `AffordabilityResult.paper_portfolio` 进入主计算结果。成交偏离过大、净值过期或对账不一致只会冻结新增订单，不会自动卖出、补仓或修改真实家庭账户。

每次三年回测保存独立运行记录。记录包含回测引擎版本、四个策略组件版本、标的池版本、行情快照 ID/数据版本、完整参数、起止日期、成本假设、结果和警告；运行指纹覆盖这些输入。输入完全相同时复用原记录，任一数据、参数或引擎版本变化都会生成新的指纹。该记录用于复盘计算口径，不代表策略通过实盘验证。

## QMT 边界与阶段门槛

`BrokerAdapter` 固定提供 `submit / cancel / query_orders / query_positions / query_cash / reconcile`。`LocalFirstBrokerGateway` 要求订单先取得本地落库 ID 和唯一 `client_order_id` 才能调用适配器；任何对账差异都冻结新增订单。当前 `QmtBrokerAdapter` 的所有方法均主动报错，未读取账号、路径或凭据。

- 二期 B：已具备手工防御资产池、季度阈值再平衡、现金/权益基准、独立交易日历、滚动样本外分段和最小方差逐信号日训练；最小方差仍仅限研究且默认关闭。
- 三期：仅在券商书面确认 QMT/XtQuant 权限后，实现只读订单、持仓、现金查询和每日对账。
- 四期：真实下单仍需人工二次确认、额度和交易时窗限制，不允许无人值守自动交易。

## Tushare Pro 配置

### 推荐：用户私有配置文件

首次在尚未配置令牌的情况下点击“用 Tushare 刷新日线”，程序会自动创建下列本机私有文件：

```text
%APPDATA%\house-planner\tushare.env
```

用文本编辑器打开它，只填写这一行并保存：

```text
TUSHARE_TOKEN=你的私有Token
```

该文件在项目目录、SQLite 家庭数据和导出文件之外；网页不会显示或保存 Token。每次刷新行情时都会重新读取它，填好后不需要重启应用。环境变量 `TUSHARE_TOKEN` 仍然优先，适合脚本或部署场景。

1. 在 Tushare Pro 官网完成注册或购买，并自行取得 Token。
2. 仅在本机用户环境变量中设置，例如 PowerShell：

   ```powershell
   [Environment]::SetEnvironmentVariable("TUSHARE_TOKEN", "你的私有令牌", "User")
   ```

3. 重开终端和后端服务，在“理财计划 → 量化定投模拟盘”点击“用 Tushare 刷新日线”。

不要把 Token 写入仓库、导出文件、家庭备注、截图或聊天消息。

## 券商确认清单

浙商证券是否支持 QMT 必须以客户经理书面答复为准。应逐项确认：个人账户能否开通 MiniQMT 和 XtQuant Python、行情与交易权限是否分别开通、是否提供仿真盘、是否覆盖场内 ETF 和港股通策略交易、资金门槛/费率/白名单规则，以及 API 是否允许本机外部 Python 连接。

若浙商不支持全部关键条件，再按同一清单评估华鑫证券等候选券商。未完成书面确认前，不实现或启用任何真实下单能力。
