# 港股通 / QDII ETF 量化定投一期

## 已实现边界

一期是“研究、提案、模拟成交、人工确认”的家庭理财辅助功能，不是自动交易系统。系统只支持用户手工维护的 ETF/基金标的池，不会推荐具体基金代码，不接收或保存任何券商、同花顺或 Tushare 凭据。

- 数据源：Tushare Pro 日频数据。令牌只能读取本机 `TUSHARE_TOKEN` 环境变量。
- 标的范围：境内 ETF、已确认合资格的港股通标的、场内跨境 QDII ETF、场外 QDII 基金。
- 交易边界：一期只有 `PaperBrokerAdapter`；`QmtBrokerAdapter` 是不可执行的占位接口。
- 风险边界：可投资现金先扣除应急金、未来 24 个月计划支出和同期重大目标资金；权益上限默认 35%，防御资产下限默认 65%。

## 稳定策略接口

提案流程拆为四个可替换但默认固定的职责：`SignalModel` 只输出月度回撤风险状态；`PortfolioConstructor` 只把权益预算分配给用户批准的标的；`PreTradeRiskManager` 负责现金、交易资格、QDII 溢价/净值可得性、单标的/单市场集中度和最大订单金额；`ExecutionPlanner` 负责下一交易日、滑点、费率和最小交易单位。每个组件都有独立版本号并进入提案和研究运行记录。

集中度不是“本月预算内的分配比例”，而是按成交后的整个模拟组合校验：分子为当前单标的或单市场持仓市值加本批次已计划买入，分母为当前模拟总权益加本次外部投入。买单以含预计费用的订单金额占用集中度额度，口径略偏保守；季度再平衡卖出不受买入集中度限制，再平衡买入会先扣除本批次计划卖出的暴露，再与其它计划买入合并校验。

一期默认只启用“月度定投 + 35/65 + 回撤降风险”。最小方差仅在 `research_strategy=min_variance` 时作为研究组件运行，默认关闭；趋势、波动率目标等策略尚未进入生产提案。

用户可把手工核实的货币/短债/债券 ETF 标记为防御资产。`PortfolioConstructor` 在权益与防御资产池之间按有效权益上限分配；没有防御标的或集中度上限未用满的部分继续保留为模拟现金，不会为了凑满 65% 自动选择产品。

季度月份仅在当前模拟权益比例偏离目标至少 5 个百分点时触发再平衡。超配持仓先生成整手卖出订单；卖出成交后，后续提案才允许使用已到账模拟现金补充低配资产。模拟现金不足或卖出超过持仓时 API 返回冲突，不会透支、截断或自动补仓。

## 策略规则

每月按沪深/港股共同可得的最近确认日线生成一次提案，所有候选快照和研究优化输入先裁剪到该共同信号日，不允许某个市场较晚的数据进入当次决策。订单优先使用行情快照中信号日后的下一共同开放日；日历接口不可用时只给出下一工作日估计并要求人工确认。模拟成交未另填日期时沿用订单计划成交日，人工回填可以晚于但不能早于计划成交日，避免成交事件倒灌到信号之前。季度偏离目标配置 5 个百分点才提示再平衡。风险篮子回撤达到 8% 时新增权益上限降为 20%；达到 12% 时暂停新增权益；达到 15% 时冻结风险资产提案并要求人工复核。该机制不能保证市场跳空时的实际最大回撤。

场内跨境 QDII ETF 必须同时拥有最新净值和收盘价：净值日龄优先按快照中的开放交易日计算，超过 3 个交易日或溢价高于策略阈值时暂停新增买入；日历不可用时才保守回退为自然日。场外 QDII 只记录净值、限购/暂停状态和人工确认，不产生交易所模拟订单。

回测使用至少三年共同可用的复权日线，并按沪深 `trade_cal`、港股 `hk_tradecal` 返回的共同开放日推进：上一个交易日生成信号，下一个所有标的均可交易的日期成交。交易日历接口无权限或不可用时，数据集会明确记录警告并回退到实际价格日期；日历开放但缺少价格的资产按停牌沿用上一估值，禁止在该日成交。停牌、申购限制、最小交易单位、费率、滑点、QDII 净值可得日期、防御资产和季度再平衡都进入撮合，不再按每 21 根日线机械压缩。输出同时包含终值、CAGR、年化波动率、最大回撤、换手率、累计费用、最低模拟现金和成交笔数；比较风险控制策略、35/65 静态配置、纯现金定投和 100% 权益固定定投。

运行记录按默认 24 个月训练窗口、12 个月测试窗口输出滚动样本外分段。默认固定规则不拟合参数，窗口只隔离评价区间；启用 `research_strategy=min_variance` 时，每个信号日只截取当日及之前最近 24 个月的权益价格重估权重，再用于下一共同可交易日，测试区间和未来数据不会进入当次拟合。该研究开关默认关闭，回测不自动选择标的，也不构成收益承诺。

模拟成交日期由适配器校验：未填写时使用订单的 `expected_trade_date`，人工回填只能是不早于计划成交日且不晚于当前日期的有效 ISO 日期。系统不允许提前确认未来成交或把成交事件倒灌到信号日前。

执行日市场门禁由模拟成交和回测撮合共用。数据集已覆盖执行日时，闭市、停牌、开放日缺价均禁止成交，`purchase_limited` 只禁止买入、不误伤卖出；数据集尚未覆盖执行日时保留人工成交确认入口。季度再平衡与普通月度投入使用同一门禁，不能借再平衡绕过申购限制。该行为对应回测引擎 `calendar-risk-v7` 和执行规划版本 `paper-lot-slippage-v3`，旧运行记录不会被新撮合口径覆盖。

## 可追溯行情数据集

每个行情版本保存数据源、接口名、抓取时间、交易日历、复权口径、价格日期、净值估值日期、净值可得日期、完整性、内容哈希和数据版本。内容哈希只覆盖可复现的数据与口径，显式排除抓取时间、内容哈希自身和由哈希派生的数据版本，因此 API 返回的完整快照可原样重放而不会产生伪新版本。`instrument_id + dataset_hash` 是不可变内容版本的唯一键：相同内容提交时只返回原记录，不能更新 `data`、`fetched_at`、`created_at` 或 `updated_at`；只有新内容哈希才新增记录、递增家庭量化数据版本并使相关计算缓存失效。这样旧回测引用的快照 ID 不会被后续刷新改写，Recorder 指纹可以稳定复现。行情 schema v3 在 QDII 公告日期缺失时保守假定净值最早于估值日后一天可得；旧 v2 快照若把估值日直接写成可得日，也按次日处理。提案、回测和事后风控共用该有效可得日，当日决策不能使用尚未发布的净值，当前日仍未可得时冻结新增。

## 模拟账户与研究记录

模拟订单只是交易意图；每个订单拥有唯一 `client_order_id`。点击模拟成交后，订单状态和成交事件必须在同一数据库事务中写入。同一订单只允许产生一条成交，重复点击返回已有成交，不重复增加持仓。月度提案采用的外部预算通过 `cash_contribution_amount` 只注入模拟账户一次，各订单至少覆盖自身预计成交所需资金，整手、费用取整和集中度约束后的未使用预算继续保留为模拟现金；只有再平衡订单的提案不注入外部资金。持仓、平均成本、市值、手续费、已实现/未实现盈亏全部由成交事件与最新行情在后端派生，不修改家庭真实现金或真实投资账户。

未成交订单取消使用 `proposed → cancel_requested → cancelled` 状态机。系统先在同一 SQLite 事务中写入 `cancel_requested` 状态和不可变订单事件，再调用适配器；只有适配器确认后才写入 `cancelled` 事件。适配器未确认时保留取消请求供重试，不能伪装成已取消；已成交、已确认、已冻结订单拒绝取消，已取消订单拒绝再次模拟成交，重复取消幂等复用原事件。

`client_order_id` 是跨本地账本与券商适配器的全局幂等键。SQLite 使用唯一表达式索引强制约束；同一订单的重复请求复用原记录，不同订单复用同一客户端订单号时 API 返回 `409`。数据库升级会在创建索引前修复缺失、非法或重复的旧订单号，并同步对应成交与取消事件中的引用，二次迁移不再改写。

模拟订单持久化格式为 schema v2。数据库启动迁移会就地为旧外部买入订单补入等于原 `order_amount` 的模拟现金投入，为旧模拟现金买入和卖出订单补 0；迁移保留订单 ID、状态和已有成交关系，并可重复执行。运行期只消费迁移后的显式字段，旧订单回退逻辑仅用于迁移前的读取保护。

`PaperPortfolioLedger` 先按 `executed_date + order_id` 重放不可变成交，再转换成现有 `MonthlyLedgerEntry → AccountSnapshotPoint → MonthlyVisualizationDetail` 数据结构，并作为 `AffordabilityResult.paper_portfolio` 进入主计算结果。每个月末的现金、持仓市值、已实现和未实现盈亏都使用当月末最近已确认价格派生；没有成交的月份也保留估值快照，不再用累计买入额减累计卖出额近似投资余额。模拟卖出和模拟现金买入按实际成交日校验当时可用持仓与现金，不能借用未来成交。模拟账户回撤按资金流中性的单位净值计算，追加资金不会掩盖既有亏损，未来行情不能进入估值或回撤；历史最大回撤达到 15% 时与成交偏离过大、净值过期、对账不一致一样，只冻结新增订单，不会自动卖出、补仓或修改真实家庭账户。

事后风控同时输出机器可读的 `PostTradeRiskIssue`：稳定 code 区分组合回撤、成交价格偏离、QDII 净值缺失/过期、卖出超过持仓和对账差异，前端只展示后端给出的严重级别、观测值、阈值与说明。对账差异会形成不可变 `BrokerReconciliationRun`，保存适配器、本地/远端订单数、持仓数、现金、状态哈希和差异清单。`mismatch` 采用锁存冻结：后续一次 `matched` 不会自动清除旧异常，必须针对原运行记录填写人工复核说明后才解除该次冻结。

每次三年回测保存独立运行记录。记录包含回测引擎版本、四个策略组件版本、标的池版本、行情快照 ID/数据版本、完整参数、起止日期、成本假设、结果和警告；运行指纹覆盖这些输入。输入完全相同时复用原记录，任一数据、参数或引擎版本变化都会生成新的指纹。该记录用于复盘计算口径，不代表策略通过实盘验证。

## QMT 边界与阶段门槛

`BrokerAdapter` 固定提供 `submit / cancel / query_orders / query_positions / query_cash / reconcile`。`LocalFirstBrokerGateway` 不再只检查 ID 非空：提交或取消前必须通过持久化校验器确认 `local_order_id + client_order_id` 与 SQLite 订单真实匹配；对账按订单状态/成交、持仓数量和现金余额生成双侧状态哈希，任何差异都锁存冻结新增订单。当前可执行的“核验模拟账本”只让 `PaperBrokerAdapter` 比较本地派生状态并保存运行记录，不读取外部账户；`QmtBrokerAdapter` 的所有方法仍主动报错，未读取账号、路径或凭据。

- 二期 B：已具备手工防御资产池、季度阈值再平衡、现金/权益基准、独立交易日历、滚动样本外分段和最小方差逐信号日训练；最小方差仍仅限研究且默认关闭。
- 三期：仅在券商书面确认 QMT/XtQuant 权限后，实现只读订单、持仓、现金查询和每日对账。
- 四期：真实下单仍需人工二次确认、额度和交易时窗限制，不允许无人值守自动交易。

## Tushare Pro 配置

### 推荐：用户私有配置文件

首次在尚未配置令牌的情况下点击“用 Tushare 刷新日线”，程序会自动创建下列本机私有文件：

```text
%APPDATA%\house-planner\tushare.env
```

用文本编辑器打开它，只填写这一行并保存：

```text
TUSHARE_TOKEN=你的私有Token
```

该文件在项目目录、SQLite 家庭数据和导出文件之外；网页不会显示或保存 Token。每次刷新行情时都会重新读取它，填好后不需要重启应用。环境变量 `TUSHARE_TOKEN` 仍然优先，适合脚本或部署场景。

1. 在 Tushare Pro 官网完成注册或购买，并自行取得 Token。
2. 仅在本机用户环境变量中设置，例如 PowerShell：

   ```powershell
   [Environment]::SetEnvironmentVariable("TUSHARE_TOKEN", "你的私有令牌", "User")
   ```

3. 重开终端和后端服务，在“理财计划 → 量化定投模拟盘”点击“用 Tushare 刷新日线”。

不要把 Token 写入仓库、导出文件、家庭备注、截图或聊天消息。

## 券商确认清单

浙商证券是否支持 QMT 必须以客户经理书面答复为准。应逐项确认：个人账户能否开通 MiniQMT 和 XtQuant Python、行情与交易权限是否分别开通、是否提供仿真盘、是否覆盖场内 ETF 和港股通策略交易、资金门槛/费率/白名单规则，以及 API 是否允许本机外部 Python 连接。

若浙商不支持全部关键条件，再按同一清单评估华鑫证券等候选券商。未完成书面确认前，不实现或启用任何真实下单能力。
