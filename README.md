# 北京家庭财务与重大消费规划工具

本项目是一个本机使用的北京家庭财务规划工具，覆盖收入、支出、购房、买车、理财、贷款、税务、账户曲线和事件时间线。前端使用 React + Vite，后端使用 FastAPI + SQLite。家庭财务数据默认保存在本机 SQLite 数据库中，不上传云端。

## 功能

- 家庭财务画像：成员、工资阶段、支出、现金账户、投资账户、公积金账户、目前贷款和购房资格参数。
- 收入阶段管理：工资就业、失业期、自由职业、养老金和手动阶段使用统一模型；阶段类型会自动配置工资社保扣缴和自由职业收入开关，也允许手动覆盖。
- 重大消费目标：买房和买车都按“目标列表、规划顺序、推荐策略、手动调整、影响预览”的交互组织。目标可新增、复制、删除、停用，并可按顺序排队、并行考虑或手动指定时间。
- 统一页面骨架：理财、养娃、税务和导出等页面也按“顶部摘要、方案/目标列表、当前配置、策略说明、影响预览”组织，减少不同页面各自一套交互。
- 策略生成：围绕目标房源、车辆需求和理财计划自动生成方案，也支持手动调整首付、贷款、购入时间、投资提取、提前还本等关键参数。
- 后端账户推演：现金余额、投资收益、贷款余额、月供、公积金账户、税费、事件时间线等核心计算以后端结果为准。
- 税务策略时间线：专项附加扣除、年终奖、个人养老金、租金/房贷利息互斥、理财收益税后口径由后端生成长期策略节点。
- 领域算法拆分：贷款、车辆、支出、养娃、税务基础、职业冲击、投资税务和幸福指数等纯计算逐步沉到 `backend/app/domain/`，便于测试和扩展。
- 响应式界面：桌面端和移动端使用同一套语义化视觉语言，错误提示会转成中文说明，图标按钮和表单控件按可访问性与对齐规则检查。
- 北京政策规则：规则包管理公积金贷款、首付比例、贷款年限、契税、社保个税资格等政策参数，便于后续扩展到其他地区政策包。商贷利率和中介费等市场报价或费用假设保留手动覆盖。
- 车贷贴息口径：主流电车“前段免息”按厂家或经销商贴息处理，贷款余额仍按全期等额本息推演；可区分合同月供、贴息后现金月供、贴息总额和家庭实际承担利息。
- 可视化故事线：房源决策表、联动月份查看、流动资产、固定资产、贷款余额、公积金账户、月现金流、税务和幸福指数解释。
- 导出方案：按当前选中的方案导出文字说明和详细表格，包含逐月账户、现金流、贷款和事件变化。

## 本地启动

### 后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
.\.venv\Scripts\python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

### 前端

```powershell
cd frontend
npm install
npm run dev -- --host 127.0.0.1
```

打开 Vite 输出的本地地址即可使用。默认前端会请求 `http://127.0.0.1:8000`。

## 首次初始化

首次启动后，系统会创建一个空白家庭，默认不添加购房或买车目标。建议按以下顺序配置：

1. 在“家庭财务”页填写家庭画像、成员名称、工资阶段、基础支出、现金账户、投资账户和公积金账户。
2. 如有需要，继续添加目前贷款、其他定时支出、老人专项扣除、职业冲击和退休养老金假设。
3. 在“购房计划”页按需添加目标房源。房源目标可以复制、停用、删除，并设置规划顺序、总价、面积、房屋性质、贷款方式和装修资金模式。
4. 在“买车计划”页按需添加车辆需求和候选车源。车辆目标同样可以复制、停用、删除，并设置购车顺序、并行或指定月份。
5. 在“理财计划”页配置投资策略，或使用系统推荐方案。
6. 点击“保存本地”，数据会写入本机 SQLite 数据库。多个房源目标会一起保存，不只保存当前选中的目标。

如果你已经在本机使用过旧版本，原有数据库不会自动清空。需要重新初始化时，请先备份后删除本机数据库文件，或设置新的 `HOUSE_PLANNER_DB` 路径。

## 数据位置

默认 SQLite 数据库位于系统应用数据目录：

- Windows: `%APPDATA%\house-planner\planner.db`
- 其他系统: `~/.house-planner/planner.db`

可以通过 `HOUSE_PLANNER_DB` 环境变量指定数据库路径。

## 开发与发布流程

新开发者或较大重构前，建议先阅读 [开发者架构说明](docs/architecture.md)。这份文档按系统边界、核心数据模型、计算请求生命周期、数据库迁移、政策包、账户/贷款/现金流和常见开发任务解释项目关系，比 README 更适合用来理解代码结构。

后端现在以 `backend/app/calculator.py` 作为计算编排入口，以 `backend/app/domain/` 承载可复用纯算法，以 `backend/app/strategies/` 承载候选策略和推荐逻辑，以 `backend/app/projection/` 承载贷款、账户和账本的逐月推演。新增功能时应先判断是“编排流程”“领域算法”“策略推荐”还是“时间线投影”：编排留在计算入口或服务层，领域算法写成可单测函数，候选策略进入策略模块，账户/贷款/现金流时间线优先进入投影模块。

推荐保持一条清晰的双轨流程：

1. 日常功能开发在 `codex/public-release` 分支进行。
2. 页面运行数据默认保存在本机 SQLite 数据库里，默认位置是 `%APPDATA%\house-planner\planner.db`，不进入 Git。
3. 代码里的默认值、测试样例和 README 使用可复现的示例数据。
4. 推送前运行：

```powershell
.\scripts\push_public.ps1
```

这个脚本会依次执行发布检查、后端测试、前端构建，并把 `codex/public-release` 推送到 GitHub `main`。

如只想快速验证发布检查：

```powershell
python scripts/privacy_scan.py --ref HEAD
python scripts/privacy_scan.py
```

本仓库还提供 `.githooks/pre-push`，用于在推送前拦截以下情况：

- 从非 `codex/public-release` 分支推送到 `main`
- 推送不符合当前发布分支基线的历史
- 待推送内容包含数据库、构建产物、依赖目录或本地配置片段

首次克隆或换机器后，可以启用本仓库的 Git hook：

```powershell
git config core.hooksPath .githooks
```

## 数据文件说明

- 不要把本机 SQLite 数据库、导出方案、截图、`.env`、日志或个人配置文件提交到 Git。
- `.gitignore` 已排除常见数据库、构建产物、虚拟环境和环境变量文件。
- 发布分支应从当前基线继续开发，避免混入旧分支历史。

## 当前建模口径

- 后端是计算真源。前端负责编辑目标和展示结果，不在本地重新推演现金、贷款、税或账户曲线。
- 买房和买车已在交互上统一为“重大消费目标”。现阶段仍沿用既有房源和车辆数据结构，后续可进一步收敛为统一 `planning_goals` 后端模型。
- 公积金贷款利率和契税来自政策规则包；房源目标里不再手动填写这类政策参数。
- 商贷利率、中介费、车贷合同利率等属于市场报价或费用假设，可以在目标里手动覆盖。
- 车辆贷款的贴息期不是合同免息期。系统按合同全期等额本息计算余额，并把贴息期内由厂家或经销商补贴的利息从家庭现金支出中扣除。

更多架构关系、数据流和扩展方式见 [docs/architecture.md](docs/architecture.md)。
